🐍 Python Class 파이썬 학습 수행평가 배포 가이드

웹 프로젝트 배포 가이드

코드 작성 → GitHub 커밋 → Vercel 배포까지의 전체 절차와 오류 대처법

0
Git 전역 설정
최초 1회 ▼

컴퓨터에서 처음 Git을 사용할 때 이름과 이메일을 등록합니다. 한 번만 하면 됩니다.

git config --global user.name "본인이름"
git config --global user.email "이메일@gmail.com"
1
프로젝트 폴더 Git 초기화 + GitHub 연결
최초 1회 ▼

GitHub에서 새 리포지토리를 만든 뒤, 로컬 폴더와 연결합니다.

git init
git remote add origin https://github.com/계정명/리포지토리명.git
GitHub → New repository → 이름 입력 → Create repository 후 진행
2
코드 작성 후 GitHub에 올리기
매번 ▼

파일을 수정할 때마다 아래 세 명령어를 순서대로 실행합니다.

git add .
git commit -m "변경 내용 설명"
git push
첫 push는 git push -u origin main 으로 실행
3
Vercel 배포
최초 1회 ▼

Vercel과 GitHub 리포지토리를 연결하면, 이후 push할 때마다 자동 배포됩니다.

  1. vercel.com → GitHub 계정으로 로그인
  2. Add New Project 클릭
  3. 리포지토리 선택 → Deploy
4
환경변수(API 키) 설정
최초 1회 ▼

API 키처럼 외부에 노출되면 안 되는 값은 Vercel 환경변수로 설정합니다.

  1. Vercel 프로젝트 → Settings → Environment Variables
  2. Name: ANTHROPIC_API_KEY / Value: API 키 입력
  3. Save 후 → Deployments → Redeploy
코드에서는 process.env.ANTHROPIC_API_KEY로 참조
5
이후 업데이트
매번 ▼

2단계와 동일합니다. push만 하면 Vercel이 자동으로 재배포합니다.

git add .
git commit -m "수정 내용"
git push
Vercel 대시보드에서 별도로 재배포할 필요 없음
!
명령어 오타 (gir, gi 등)
▼
'gir' 용어가 cmdlet... 인식되지 않습니다

원인: git 명령어 철자 오류

대처법

명령어를 다시 확인하고 정확히 입력하세요. git은 g-i-t 세 글자입니다.

!
GitHub 리포지토리를 찾을 수 없음
▼
remote: Repository not found. fatal: repository '...' not found

원인: GitHub에 리포지토리를 먼저 만들지 않고 push했거나, 주소가 잘못됨

대처법

GitHub에서 리포지토리를 먼저 생성한 뒤 push하세요. 주소도 정확한지 확인하세요.

git remote -v  # 현재 연결된 주소 확인
!
push 거부 (원격에 로컬에 없는 커밋 존재)
▼
Updates were rejected because the remote contains work that you do not have locally

원인: GitHub에서 직접 파일을 생성했거나 다른 곳에서 push한 경우 (예: README 자동 생성)

대처법
git pull --rebase origin main
git push
!
파일이 빈 채로 배포됨
▼
기능이 동작하지 않음 / API 오류 발생

원인: 파일을 만들었지만 git add를 하지 않아 빈 파일이 GitHub에 올라감

대처법
git status          # 스테이징 안 된 파일 확인
git add 파일명
git commit -m "파일 추가"
git push
예방: push 전에 항상 git status로 확인
!
API를 프론트엔드에서 직접 호출
▼
CORS 오류 또는 API 키 노출 위험

원인: HTML/JS에서 외부 API를 직접 호출하면 브라우저 보안 정책(CORS)에 막히고 API 키가 노출됨

대처법
// 잘못된 방법
fetch('https://api.anthropic.com/v1/messages', ...)

// 올바른 방법: 서버 프록시를 거침
fetch('/api/chat', ...)
api/ 폴더에 서버 함수를 만들고 API 키는 환경변수로 관리
!
잘못된 AI 모델명
▼
API 오류 / 피드백이 오지 않음

원인: 존재하지 않는 모델 ID를 사용

대처법
// 올바른 모델명 예시
model: 'claude-3-5-sonnet-20241022'
Anthropic 공식 문서에서 현재 사용 가능한 모델명 확인
!
환경변수 추가 후 여전히 인증 오류
▼
API 인증 실패 (401 Unauthorized)

원인: 환경변수는 다음 배포부터 적용됨. 추가 후 Redeploy를 하지 않으면 반영 안 됨

대처법
  1. Vercel → Deployments 탭 클릭
  2. 최신 배포 옆 ... → Redeploy
